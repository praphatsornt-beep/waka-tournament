## 1. ลูกค้า (Customer)

### 1.1 สั่งซื้อสินค้า
```
เปิดลิงก์ → เลือกสินค้า + จำนวน → กรอกข้อมูล → แนบสลิป → ส่งออเดอร์
```

| ขั้นตอน | ลิงก์ |
|---------|-------|
| สั่งซื้อ (ทุกสินค้า) | https://liff.line.me/2010457385-UpJLXxJ0 |
| สั่งซื้อ (แยกสินค้า) | https://liff.line.me/2010457385-UpJLXxJ0/{slug} เช่น `/SL02` |
| ดูลิงก์แยกสินค้าทั้งหมด | เปิดหน้า Menu → ส่วน "ลิงก์สั่งซื้อแยกสินค้า" |

- ระบบตรวจสลิปอัตโนมัติ (SlipOK → Claude AI fallback)
- จำกัดจำนวนสั่งซื้อตาม `limit_box` / `limit_pack` ใน `_catalog`

### 1.2 รับสินค้า
```
ได้รับ LINE แจ้ง "สินค้าพร้อมรับ" → ไปสาขา → พนักงานส่งมอบ → กดยืนยันรับของ
```

| ขั้นตอน | ลิงก์ |
|---------|-------|
| ดูสถานะ / กดยืนยันรับของ | ลิงก์จาก LINE notification (confirm.html?order=XXXXXX) |

## 2. พนักงานคลังกลาง (กัน + จั้ม + น้องใหม่)

### เมนูหลัก
| ลิงก์ | หน้าที่ |
|-------|---------|
| **เมนูรวม** | https://waka-liff.vercel.app/menu.html |
| **คลังกลาง** | https://waka-liff.vercel.app/warehouse.html |

### 2.1 รับสินค้าเข้าคลังกลาง (กัน / จั้ม)
```
สินค้ามาถึงคลัง → เช็คของ → อัปเดตสต็อกใน Google Sheets
```

| ขั้นตอน | ที่ไหน |
|---------|--------|
| เพิ่มจำนวนสต็อก | Google Sheets → tab `stock` → แก้ qty_box / qty_pack |
| เพิ่มสินค้าใหม่ | Google Sheets → tab `_catalog` (ข้อมูลสินค้า) + tab `stock` (สต็อก) |
| ตั้ง limit ปล่อยขาย | Google Sheets → tab `_catalog` → คอลัมน์ J (limit_box) / K (limit_pack) |
| ล้าง cache หลังแก้ | GAS editor → รัน `clearCache()` |

### 2.2 สร้างล็อตส่งสาขา (กัน / จั้ม)
```
เปิดหน้าคลังกลาง → ดูสรุปออเดอร์แต่ละสาขา
→ เลือกสาขา → กำหนดจำนวนส่ง + จำนวนเผื่อ → สร้างล็อต
→ ตัดสต็อกกลางอัตโนมัติ
```

| ลิงก์ | หน้าที่ |
|-------|---------|
| **คลังกลาง → แท็บ "สรุปสาขา"** | ดูจำนวนสินค้าที่ต้องส่งแต่ละสาขา |
| **คลังกลาง → แท็บ "สร้างล็อต"** | กำหนดจำนวนส่ง + เผื่อ → สร้างล็อต |
| **คลังกลาง → แท็บ "ประวัติ"** | ดูรายการล็อตที่เคยส่ง + สถานะ |
| **คลังกลาง → แท็บ "สต็อก"** | ดูสต็อกกลาง + สต็อกแต่ละสาขา |

### 2.3 นับของ & จัดส่งไปสาขา (น้องใหม่)
```
กัน/จั้ม สร้างล็อตแล้ว → น้องใหม่ดูรายการในแท็บ "ประวัติ"
→ นับของตามรายการ → จัดส่งให้สาขา
```

| ลิงก์ | หน้าที่ |
|-------|---------|
| **คลังกลาง → แท็บ "ประวัติ"** | ดูรายการล็อตที่ต้องส่ง (สถานะ: จัดส่ง) |

## 3. พนักงานสาขา (ต้นสัก / เมืองทอง / ศรีนครินทร์)

### ลิงก์ประจำสาขา
| สาขา | ลิงก์ |
|------|-------|
| **ต้นสัก** | https://waka-liff.vercel.app/branch.html?branch=ต้นสัก |
| **เมืองทอง** | https://waka-liff.vercel.app/branch.html?branch=เมืองทอง |
| **ศรีนครินทร์** | https://waka-liff.vercel.app/branch.html?branch=ศรีนครินทร์ |
| **จัดการออเดอร์** | https://waka-liff.vercel.app/staff.html |

### 3.1 รับของจากคลังกลาง
```
ล็อตมาถึงสาขา → เปิดหน้าคลังกลาง > ประวัติ → กด "ยืนยันรับของ"
→ เช็คว่าของครบตามรายการ
→ สต็อกสาขาเพิ่มอัตโนมัติ
→ LINE แจ้งลูกค้าทุกคนที่มีออเดอร์ที่สาขานี้: "สินค้าพร้อมรับ"
```

| ลิงก์ | หน้าที่ |
|-------|---------|
| **คลังกลาง → แท็บ "ประวัติ"** | กด "ยืนยันรับของ" เมื่อของมาถึง |
| **หน้าสาขา → แท็บ "สต็อก"** | ดูสต็อกสินค้าคงเหลือของสาขา |

### 3.2 ส่งมอบลูกค้า
```
ลูกค้ามาหน้าร้าน → เปิดหน้าจัดการออเดอร์ → ค้นหาชื่อ/เบอร์/order ID
→ กด "ส่งมอบสินค้า" → ตัดสต็อกสาขาอัตโนมัติ
→ LINE แจ้งลูกค้า: "กรุณากดยืนยันรับของ"
→ ลูกค้ากดยืนยัน → สถานะเป็น "รับแล้ว" ✅
```

| ลิงก์ | หน้าที่ |
|-------|---------|
| **จัดการออเดอร์** | https://waka-liff.vercel.app/staff.html |

### 3.3 จัดส่งพัสดุ (เฉพาะออเดอร์ "จัดส่ง")
```
ออเดอร์ "จัดส่ง" → เปิดหน้าจัดการออเดอร์ → ค้นหาออเดอร์
→ กด "จัดส่งพัสดุแล้ว" → LINE แจ้งลูกค้ากดยืนยัน
```

## 4. คนคุมบัญชี (Finance)

### 4.1 ตรวจสลิปที่มีปัญหา
```
ได้รับ LINE แจ้ง ⚠️ สลิปมีปัญหา → เปิด Streamlit → ตรวจสลิป
→ เช็คแอปธนาคาร → แก้สถานะในชีตเป็น "ยืนยัน"
```

| ลิงก์ | หน้าที่ |
|-------|---------|
| **Streamlit Admin** | https://waka-tournament-e6wsqmhuhhexratyiub65f.streamlit.app |
| **Google Sheets** | tab `orders` → กรองคอลัมน์ `slip_status` |

**ตั้งค่า:** ใส่ LINE User ID ของคนตรวจบัญชีใน `_config` → key: `finance_line_id`

**สถานะสลิปที่ต้องตรวจ:**
| สถานะ | ความหมาย | สิ่งที่ต้องทำ |
|-------|----------|-------------|
| ยืนยัน | ผ่านอัตโนมัติ | ไม่ต้องทำอะไร ✅ |
| รอตรวจ | อ่านสลิปไม่ได้ | เช็คสลิปใน Drive + แอปธนาคาร → แก้สถานะเป็น "ยืนยัน" |
| รอตรวจเพิ่ม | Claude อ่านได้แต่ไม่ 100% ตรง | เช็คแอปธนาคาร → แก้สถานะ |
| ยอดไม่ตรง | สลิป < ยอดออเดอร์ | ติดต่อลูกค้า หรือเช็คว่าโอนครบ |
| สลิปซ้ำ | เลขอ้างอิงซ้ำกับออเดอร์อื่น | ติดต่อลูกค้าให้ส่งสลิปใหม่ |
| บัญชีไม่ตรง | โอนผิดบัญชี | ตรวจสอบในแอปธนาคาร |
| สงสัยปลอม | Claude คิดว่าสลิปตัดต่อ | ตรวจสอบด้วยตา + แอปธนาคาร |
| ไม่มีสลิป | ไม่ได้แนบสลิป | ติดต่อลูกค้า |

**วิธีแก้สถานะ:** แก้ในชีตตรงๆ คอลัมน์ `slip_status` เป็น "ยืนยัน"

### 4.2 ดูรายงานยอดขาย / กำไร

| ลิงก์ | หน้าที่ |
|-------|---------|
| **รายงานยอดขาย** | https://waka-liff.vercel.app/report.html |

- ยอดขาย / ต้นทุน / กำไร
- แยกตาม: สาขา / สินค้า / รายวัน
- เฉพาะออเดอร์ที่ slip_status = "ยืนยัน"

## 5. ตั้งค่าสินค้า (_catalog)

| คอลัมน์ | ใช้ทำอะไร |
|---------|-----------|
| A: name | ชื่อสินค้า |
| B: category | หมวดหมู่ |
| C: price_box | ราคา Box |
| D: price_pack | ราคา Pack (ว่าง = ไม่มี Pack) |
| E: active | TRUE/FALSE เปิด-ปิดสินค้า |
| F: image_url | ลิงก์รูป Google Drive |
| G: cost_box | ต้นทุน Box (สำหรับรายงานกำไร) |
| H: cost_pack | ต้นทุน Pack |
| I: slug | URL ย่อ เช่น SL02 → /SL02 |
| J: limit_box | จำนวน Box ที่ปล่อยขาย (ว่าง = ไม่จำกัด) |
| K: limit_pack | จำนวน Pack ที่ปล่อยขาย (ว่าง = ไม่จำกัด) |

**หมายเหตุ:** หลังแก้ราคา/limit → รัน `clearCache()` ใน GAS editor เพื่ออัปเดตทันที (หรือรอ 5 นาที)

## 6. สรุปคนรับผิดชอบ

| บทบาท | คน | ทำอะไร | เข้าที่ไหน |
|-------|-----|--------|-----------|
| คลังกลาง — รับเข้า/ตั้ง limit | กัน, จั้ม | รับสินค้า, อัปเดตสต็อก, ตั้ง limit | Google Sheets (stock, _catalog) |
| คลังกลาง — สร้างล็อต | กัน, จั้ม | กำหนดจำนวนส่งแต่ละสาขา | warehouse.html |
| คลังกลาง — นับ/จัดส่ง | น้องใหม่ | นับของตามล็อต, ส่งไปสาขา | warehouse.html (แท็บประวัติ) |
| สาขา — รับของ | พนักงานสาขา | กดยืนยันรับของ, เช็คสต็อก | warehouse.html + branch.html |
| สาขา — ส่งมอบลูกค้า | พนักงานสาขา | ค้นหาออเดอร์, กดส่งมอบ | staff.html |
| บัญชี — ตรวจสลิป | คนคุมบัญชี | ตรวจสลิปมีปัญหา, แก้สถานะ | Streamlit + Google Sheets |
| บัญชี — รายงาน | คนคุมบัญชี | ดูยอดขาย/กำไร | report.html |
